# Урок 3 Без Фреймворка: Чтение Одного Ресурса И Списка На Чистой Java

## Зачем нужен этот файл

Он показывает, как те же сценарии выглядели бы без Spring MVC.

То есть без:

- `@PathVariable`
- `@RequestParam`
- автоматического маппинга параметров
- автоматической сериализации ответа

---

## Что пришлось бы делать вручную

Для `GET /api/tasks/1`:

- разбирать URL строкой
- вырезать id вручную
- преобразовывать строку в `Long`

Для `GET /api/tasks?type=SHELL_COMMAND`:

- разбирать query string вручную
- извлекать параметры вручную

И потом:

- вручную собирать JSON
- вручную отправлять HTTP response

---

## Главный вывод

Spring не “рисует магию”.

Он убирает большое количество повторяющейся web-инфраструктуры:

- path parsing
- query parsing
- вызов нужного метода
- serialization

Поэтому профессиональный Spring-код короче и яснее.
