# Урок 2. Типичные Ошибки

## Ошибка 1. Принимать `Map<String, Object>` вместо request DTO

Это делает входной контракт неявным и хрупким.

---

## Ошибка 2. Возвращать `200 OK`, не думая о semantics

Для успешного create endpoint-а правильнее явно использовать `201 Created`.

---

## Ошибка 3. Смешивать request и response модель в один класс без причины

Вход и выход — это разные роли.

---

## Ошибка 4. Класть create logic прямо в controller

Controller должен принимать HTTP и делегировать use case.

---

## Ошибка 5. Не понимать, что `@RequestBody` делает реально

Эта аннотация не “украшает параметр”, а подключает маппинг JSON body в Java-объект.
