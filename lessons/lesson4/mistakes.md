# Урок 4. Типичные Ошибки

## Ошибка 1. Использовать `POST` там, где нужен `PUT`

Если задача уже существует и ты меняешь ее по id, это уже update use case.

---

## Ошибка 2. Возвращать строку после delete

`"Deleted successfully"` — слабый API-ответ, если лучше подходит правильный HTTP status.

---

## Ошибка 3. Обновлять через `Map<String, Object>`

Такой входной контракт слишком неявный и хрупкий.

---

## Ошибка 4. Смешивать update/delete logic с HTTP-слоем

Controller должен принимать запрос и делегировать use case.

---

## Ошибка 5. Не видеть разницы между create, update и delete

Это три разных сценария и три разных формы API-поведения.
