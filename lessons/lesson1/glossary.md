# Урок 1. Словарь Терминов

## Как использовать этот словарь

Это не полный словарь Spring.

Это компактный набор терминов, без которых нельзя понять первый урок.

Если во время чтения материалов тебе встретилось слово, смысл которого ты не можешь объяснить своими словами, вернись сюда.

---

## Application

Приложение, которое мы запускаем как backend-сервер.

В контексте урока это Spring Boot приложение, которое принимает HTTP-запросы.

---

## Backend

Серверная часть системы.

Она принимает запросы, выполняет логику и возвращает ответы.

---

## Bean

Объект, которым управляет Spring container.

Spring создает bean, хранит его и может передавать его другим bean-ам.

---

## Constructor Injection

Способ передачи зависимостей через конструктор класса.

В уроке это основной правильный способ DI.

---

## Controller

Класс web-слоя, который принимает HTTP-запрос и передает работу дальше.

Controller не должен становиться хранилищем всей логики приложения.

---

## Dependency

Объект, который нужен другому объекту для работы.

Например, `TaskExecutionController` зависит от `TaskRunService`.

---

## Dependency Injection

Подход, при котором класс не создает свои зависимости сам, а получает их извне.

В Spring это обычно делает container.

---

## DTO

Data Transfer Object.

Объект, который используется для передачи данных между слоями или наружу через API.

В уроке `OrchestratorStatusResponse` — это DTO.

---

## Endpoint

Конкретная HTTP-точка входа в приложение.

Обычно включает:

- URL
- HTTP method
- код, который будет вызван

---

## Framework

Готовая программная платформа, которая берет на себя часть инфраструктурной работы и задает правила организации кода.

Spring — это framework.

---

## HTTP

Протокол, по которому клиент и сервер обмениваются запросами и ответами.

---

## HTTP Method

Тип HTTP-запроса.

Например:

- `GET`
- `POST`
- `PUT`
- `DELETE`

В первом уроке нас интересует `GET`.

---

## JSON

Текстовый формат передачи данных.

Именно в JSON backend обычно возвращает ответы клиенту.

---

## Lombok

Java-библиотека, которая генерирует шаблонный код по аннотациям.

В уроке она помогает не писать вручную конструкторы и логгер.

---

## `main()`

Точка входа Java-приложения.

В Spring Boot через `main()` запускается все приложение.

---

## Object Graph

Набор объектов приложения и связей между ними.

Когда Spring делает DI, он фактически собирает этот граф объектов.

---

## `record`

Специальная конструкция Java для компактного объявления неизменяемых объектов данных.

Очень удобна для DTO.

---

## Request

Запрос клиента к серверу.

В уроке:

`GET /api/orchestrator/status`

---

## Response

Ответ сервера на запрос клиента.

В уроке ответ содержит JSON со статусом оркестратора.

---

## ResponseEntity

Класс Spring, который позволяет явно управлять HTTP-ответом:

- статусом
- телом
- заголовками

---

## Route

Маршрут, по которому HTTP-запрос связывается с обработчиком.

В Spring route обычно задается через mapping-аннотации.

---

## Service

Класс прикладного слоя, где лежит use case или прикладная логика.

Он не принимает HTTP напрямую, а вызывается из controller.

---

## Spring

Общее название экосистемы и framework-платформы для Java-приложений.

---

## Spring Boot

Надстройка над Spring, которая упрощает запуск и настройку приложения.

Она сокращает объем ручной конфигурации.

---

## Spring Web

Часть экосистемы Spring, которая отвечает за HTTP-слой:

- controller
- маршруты
- обработку запросов
- отправку ответов

---

## Status Code

HTTP-код ответа сервера.

Например:

- `200 OK`
- `404 Not Found`
- `500 Internal Server Error`

---

## Use Case

Отдельное прикладное действие системы.

В уроке use case:

“получить статус оркестратора”.

---

## Wiring

Процесс связывания объектов приложения друг с другом.

В чистой Java версии wiring часто делается вручную.
В Spring это обычно делает container.

---

## Главное различие, которое нужно запомнить

Новичок часто видит только код метода.

Инженер видит:

- роль класса
- место класса в архитектуре
- зависимости
- контракт ответа
- путь запроса через систему

Первый урок и нужен для перехода от первого взгляда ко второму.
