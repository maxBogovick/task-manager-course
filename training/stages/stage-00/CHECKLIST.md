# Чек-Лист Stage 00

## Проверка результата

- проект запускается;
- тест `mvn test` проходит;
- понятно, где находится точка входа приложения;
- понятно, где находится минимальный HTTP endpoint;
- понятно, какие части проекта относятся к инфраструктурному каркасу.

## Команды

```bash
mvn test
mvn spring-boot:run
```

## Вопросы для самопроверки

1. Какую роль выполняет `TaskManagerApplication`?
2. Почему `HealthController` полезен на этом этапе?
3. Почему здесь еще нет доменных сущностей?
4. Какие части проекта можно считать инфраструктурным минимумом?

## Что должно остаться после этапа

- понимание стартовой структуры проекта;
- уверенность в запуске приложения;
- понимание, куда будет добавляться доменная логика на следующем этапе.
