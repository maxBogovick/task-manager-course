# Рубрика Оценки

Используй эту рубрику после каждого lab.

## 5 - Отлично

- студент сам нашел точку входа;
- построил корректную цепочку вызовов;
- назвал root cause, а не поверхностный симптом;
- написал тест, который падает до фикса;
- сделал минимальный фикс;
- объяснил остаточный риск.

## 4 - Хорошо

- студент понял основной поток;
- причина названа почти точно;
- тест или проверка покрывает главный сценарий;
- фикс небольшой;
- есть небольшие пробелы в объяснении edge cases.

## 3 - Достаточно

- студент дошел до рабочего фикса;
- часть расследования сделана с подсказками;
- тест слабый или проверяет только happy path;
- причина объяснена неполно;
- есть риск случайного изменения соседнего поведения.

## 2 - Нужно Переделать

- студент начал менять код без диагностики;
- причина не доказана;
- тест отсутствует;
- фикс широкий или архитектурно случайный;
- студент не может объяснить, почему поведение исправлено.

## 1 - Не Засчитано

- код переписан без связи с симптомом;
- существующие tests broken;
- публичный API изменен без причины;
- отчет отсутствует;
- студент не может пройти цепочку вызовов.

## Быстрый чек преподавателя

Перед оценкой спроси:

1. Где начинается сценарий?
2. Где находится бизнес-решение?
3. Где состояние пишется в БД?
4. Какой тест ловит баг?
5. Почему фикс не шире необходимого?
