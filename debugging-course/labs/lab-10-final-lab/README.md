# Lab 10 - Final Lab

## Цель

Проверить самостоятельность: студент получает только симптом и должен провести расследование end-to-end.

## Симптом

Выбери один из вариантов:

1. `POST /api/tasks` принимает невалидный config для нового executor.
2. Retry создает execution с неправильным attempt.
3. Scheduled task запускается повторно чаще ожидаемого.
4. Timeout execution позже становится `FAILED`.
5. Новый executor не появляется в `/api/tasks/types`.

## Условия

- преподаватель выбирает один симптом;
- студент не получает список файлов;
- студент обязан вести debugging report;
- студент обязан предложить тест;
- студент обязан объяснить минимальный фикс.

## Задание

1. Запиши симптом.
2. Найди точку входа.
3. Построй цепочку вызовов.
4. Сформулируй минимум 3 гипотезы.
5. Проверь гипотезы от самой дешевой к самой дорогой.
6. Напиши regression test.
7. Сделай минимальный фикс.
8. Запусти проверку.
9. Заполни итоговый отчет.

## Ограничения

- не использовать подсказки из предыдущих labs первые 30 минут;
- не менять больше 3 production files без согласования;
- не менять публичный API без отдельного решения;
- не принимать fix без объяснения root cause.

## Критерии оценки

- 30% - правильно найден поток выполнения;
- 25% - правильно названа причина;
- 20% - тест действительно ловит баг;
- 15% - фикс минимальный;
- 10% - студент назвал остаточный риск.

## Эталонное рассуждение

У финального lab нет одного эталонного diff. Эталон - процесс:

```text
symptom -> entry point -> call chain -> hypotheses -> failing test -> minimal fix -> verification -> risk
```

Если студент пропускает середину и сразу меняет код, lab не засчитывается даже при зеленых тестах.

## Проверка

```bash
mvn test
```
