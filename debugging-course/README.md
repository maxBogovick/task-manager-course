# Debugging Course

Этот курс учит студента работать с уже существующим кодом в `src`: читать его, строить гипотезы, находить дефекты, писать регрессионные тесты и делать минимальные исправления.

Это не курс по синтаксису Java и не курс "переписать проект заново". Главный навык: войти в рабочую систему, понять ее правила и изменить ровно то, что нужно.

## Для кого

- студент уже знает базовую Java;
- студент понимает, что такое HTTP, JSON, SQL и тест;
- студент может запустить `mvn test`;
- студенту нужно научиться читать Spring Boot backend как production-код.

## Как проходить

1. Прочитать [Правила курса](./COURSE_RULES.md).
2. Прочитать [Архитектурную карту](./architecture/README.md).
3. Делать labs строго по порядку.
4. В каждом lab сначала написать наблюдения и гипотезу.
5. Перед фиксом написать или обновить тест.
6. После фикса объяснить, почему исправление минимальное.

## Команды

```bash
mvn test
mvn test -Dtest="TaskManagerApplicationTests#shouldCreateTaskDefinition"
```

Тесты используют PostgreSQL через Spring Boot Docker Compose support и `compose-test.yaml`.

## Что считается успехом

Студент умеет:

- найти точку входа по URL;
- пройти путь Controller -> Service -> Repository -> Entity;
- объяснить разницу между `TaskDefinition` и `TaskExecution`;
- найти, где меняется статус выполнения;
- воспроизвести баг тестом;
- исправить баг без переписывания архитектуры;
- объяснить риск изменения.
